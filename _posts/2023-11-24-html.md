## HTMLの基礎知識

- 要素の中には、タグを省略できるものがある
- 空要素は、＜〇〇＞と書いても＜〇〇 /＞と書いてもOK
- DOCTYPE宣言の基本形は<!DOCTYPE html>だが、これ以外も指定可能

※空要素：タグで囲うコンテンツ(要素内容)がなく、その存在を開始タグだけで示す要素
例）img要素、br要素、input要素、meta要素、link要素

## グローバル属性

- class属性の値には、種類や分類を示す名前を指定する
- id属性の値には、他の要素と重複しない固有の名前を指定する
- HTML5以降では、data-○○○の書式で独自の属性を追加できる

### 列挙属性／列挙型属性(Enumerated Attribute)

あらかじめ決められたキーワードの中から値を選択する方針の属性のことを「列挙属性」または「列挙型属性」という。キーワードは大文字で書いても小文字で書いても文法的に問題ない。

列挙属性の中には値として空文字を指定できるものもあり、「translate=""」を指定すると「translate="yes"」と同じ結果になる。同様に「spellcheck=""」は「spellcheck="true"」、「contenteditable=""」は「contenteditable="true"」、「hidden=""」は「hidden="hidden"」と同じ結果になる。

### 論理属性／論理型属性(Boolean Attribute)

通常、HTMLの属性は「属性名="値"」の書式で指定するが、「属性名」だけで指定できる属性のことを「論理属性」または「論理型属性」という。
論理属性は、属性を指定すると値が「true」になり、指定しないと「false」の状態となる。
「属性名だけで指定すると、その属性名のとおりになる属性」が論理属性。

論理属には値を指定するもできる。その場合は、「属性名=""」のように値を空にするか「属性名="属性名"」のように指定する必要がある。

## 全体構造とメタ情報

- title要素はなくても良いケースがある
- meta要素のcharset属性にはUTF-8しか指定できない
- link要素で外部スタイルシートを読み込む際のtype属性のデフォルト値は「text/css」

### プラグマディレクティブ／プラグマ指示子

プラグマディレクティブとは、HTML文書の状態や挙動を指示する命令のこと。たとえば、ブラウザへの再読み込みや他文書への移動の命令、デフォルトのスタイルシート言語やスクリプト言語の設定、文字コードの宣言(HTML5より前の指定方法)などがこれに該当する。meta要素にhttp-equiv属性が指定されているとき、そのmeta要素はプラグマディレクティブとなる。

使用例

```html
<head>
  <meta http-equiv="refresh" content="10; URL=page2.html">
  <!-- 10秒後にpage2.htmlに移動 -->
</head>
```

## セクションと基本構造
- h1~h6要素は「セクションの見出し」をあらわす
- アウトラインアルゴリズムとその関連使用は2022年7月に仕様から削除された
- hgroup要素は、見出し(h1~h6要素)とサブタイトルや副題(p要素)をグループ化する要素

### セクションと見出しの関係

セクショニングコンテンツに分類される4つの要素(artcle要素・section要素・aside要素・nav要素)はそれぞれ、その範囲が1つのセクションであることを示す。HTMLのセクションとは、「見出しとそれに対応する文書全体を含む範囲」のこと。

ただし、見出しとセクショニングコンテンツは常に1対1のセットで使用する決まりになっているわけではなく、「見出しのないセクショニングコンテンツ」も存在すれば、「セクショニングコンテンツのない見出し」も存在する。

## テキスト
- mark要素は、注目してもらうために目立たせている部分をあらわす
- data要素は、機械読取が可能なデータを属性値として提供する要素
- time要素は、data要素を日時に特化させた要素


## リスト
- ul要素・ol要素・dl要素の内容は空でもOK
- ol要素は、reversed属性で逆順にできる
- dl要素で用語の定義をする場合、dfn要素も必要になる

## ルビ
- ルビ関連要素には、ruby要素・rt要素・rp要素の3種類がある
- rt要素とrp要素の終了タグは省略可能
- HTML5で定義されていたrb要素とrfc要素は削除された

rubyはフリガナのこと。

rt要素の「rt」は「ruby text」の略で、ルビとして小さい文字で表示させる

rp要素の「rp」は「ruby parentheses(パーレン＝丸括弧)」のことで、ルビ未対応のブラウザでルビが普通サイズのテキストで表示される際に使用する。

```html
<ruby>亜米利加<rp>（</rp><rt>アメリカ</rt><rp>）</rp></ruby>
```

## 画像・動画・音声
- img要素のalt属性は、特別なケースにおいては省略できる
- 複数の候補画像の中から最適なものを表示させる機能がある
- width属性とheight属性にパーセント値は指定できない

img要素のsrcset属性の値にはURLと記述子(2x, 3xなど)を空白文字で区切ったものを、カンマ区切りで複数指定できる。srcset属性に指定可能な記述子には、 **ピクセル密度記述子** と **幅記述子** の2種類ある。

### ピクセル密度記述子(pixel density desciptor)

srcset属性でURLに続けて指定する「1.5x」「2x」「3x」などの記述子をピクセル密度記述子という。これは、そのURLの画像が「ピクセル密度が何倍のデバイス向け」である野かを示すもので、数値(浮動小数点)の直後に小文字の「x」をつけてあらわす。主に画像の表示サイズが固定である場合に使用される。srcset属性で記述子を省略すると、「1x」が指定されてる場合と同様に処理される。

### 幅記述子(width desciptor)

srcset属性でURLに続けて指定する「400w」「800w」「1600w」などの記述子を幅記述子という。これは、そのURLの画像の実際の幅(ピクセル数)を示すもので、整数の直後に小文字の「w」をつけて表す。この記述子は、主に画像の表示サイズが状況によって変化する場合に使用されるものでブラウザはサイズの異なるこれらの画像の中から、表示させる幅やそのデバイスのピクセル密度、および画像の拡大縮小状況などに合わせて最適な画像だけをロードする。

### size属性

size属性には、条件を指定せずに、CSSの単位をつけた表示幅だけを指定することもできる。

次の例で使用されている単位「vw」は「ビューポートの幅(viewportのwidth)」に対するパーセンテージを表すので、100vwはgっ増の表示幅を表示領域全体の幅100%にする指定となる。そしてその幅やピクセル密度などに応じて、幅が400ピクセル・800ピクセル・1600ピクセルの画像のうち、最適なものだけがロードされ表示される。

使用例
```html
<img size="100vw" srcset="small.jpg 400w, medium.jpg 800w, large.jpg 1600w" src="small.jpg" alt="">
```

条件ごとの表示幅を複数指定する際には、()内に入れた条件式と表示幅を空白文字で区切ったものをカンマ区切りで必要なだけ指定できる。

次のソースコードでは、ビューポートの幅が30em以下だった場合は100vw、30emより大きく50em以下だったら50vw、50emより大きい場合は33vwｋら60px引いた幅となる。

使用例
```html
<img size="(max-width: 30em) 100vw, (max-width: 50em) 50vw, calc(33vw - 60px)" srcset="small.jpg 400w, medium.jpg 800w, large.jpg 1600w" src="small.jpg" alt="">
```

また、size属性を指定したら、必ずsrcset属性も指定する必要がある。そして、srcset属性のすべてのURLには幅記述子をつける。

### picture要素

候補画像を示すsource要素とimg要素をとりまとめるための要素。

source要素がpicture要素内でも使用可能。

示された条件に最初に合致した画像だけがロードされ表示される。

```html
<picture>
  <source media="(min-width: 50em)" srcset="large.jpg">
  <source media="(min-width: 30em)" srcset="medium.jpg">
  <img src="small.jpg" alt="">
</picture>
```

### source要素

picture要素・video要素・audio要素の内部において、サイズやファイル形式などが異なる代替データを指定するための要素。

picure要素内のsource要素に指定できる属性と、video要素とaudio要素内のsource要素に指定できる属性が違う。

- picure要素内のsource要素に指定できる属性

| 属性名 | 値の示すもの |　指定可能な値　|
| ------ | ------ | ------ |
| media | 画像の使用条件 |　メディアクエリ　|
| srcset | 候補画像 |　URLと記述子　|
| sizes | 使用条件と画像の表示幅 |　メディアクエリと表示幅　|
| type | 組み込む画像の種類 |　MIMEタイプ　|
| width | 幅 |　整数(負の値は指定不可)　|
| height | 高さ |　整数(負の値は指定不可)　|

- video要素とaudio要素内のsource要素に指定できる属性

| 属性名 | 値の示すもの |　指定可能な値　|
| ------ | ------ | ------ |
| src | データのアドレス |　URL(空文字は不可)　|
| type | 組み込むデータの種類 |　MIMEタイプ　|

picute要素内のsource属性には、srcset属性を必ず指定する必要がある

次の例では、ビューポートの幅が500px以下の時にはピクセル密度に合わせてphone.jpg(1x用)またはphone-retina.jpg(2x用)のいずれかが表示される。srcset属性やsizes属性の使用方法はimg要素の場合と同じ。

```html
<picture>
  <source media="(max-width: 500px)" srcset="phone.jpg, phone-retina.jpg 2x">
  <img src="pc.jpg" srcset="pc-retina.jpg 2x" alt="">
</picture>
```

次の例では、条件を示すmedia属性を使用せずにtype属性を使用している。

```html
<picture>
  <source srcset="pic.apng" type="image/apng">
  <source srcset="pic.webp" type="image/webp">
  <img src="pic.jpg" alt="">
</picture>
```

### track要素

動画や音声データ(video要素またはaudio要素)に同期した外部テキスト・トラック(字幕のデータなど)を指定する場合に使用する

```html
<video src="trip.ogv" controls>
  <track kind="subtitles" src="trip.ja.vtt" srclang="ja" label="日本語">
  <track kind="subtitles" src="trip.en.vtt" srclang="en" label="English">
</video>
```

track要素は、video要素およびaudio要素の要素内容として、source要素よりも後、その他の要素よりも前に配置する必要がある。


### embed要素

外部アプリケーションやインタラクティブなコンテンツなどを、プラグインを使って組み込む際の要素。

```html
<embed src="game.swf" type="application/x-shockwave-flash" width="500" height="500" quality="high">
```

### map属性

イメージマップを定義するための要素。イメージマップとは、複数のリンクを入れることができる画像のようなもの。

```html
<img src="map.png" usemap="#navbar" alt="ナビゲーションバー">
...
<map name="navber">
  <area href="prev.html" alt="前ページ" shap="rect" coords="100,10,170,30">
  <area href="next.html" alt="次ページ" shap="rect" coords="200,10,270,30">
</map>
```

### area要素

イメージマップでリンクにする領域を定義する空要素。

反応する領域は、shape属性で指定する。値はキーワードで指定し、rectは四角形、circleは円、polyは多角形、defaultは画像全体となる。

### object要素

さまざまな形式の外部データを読み込むための要素。

6.HTMLの要素の中には、「開始タグ・終了タグとも省略できないもの」「開始タグ・終了タグとも省略できるもの」「終了タグのみ省略できるもの」の3種類がある。

HTMLの全要素のうち8割以上は「開始タグ・終了タグとも省略できないもの」に該当、html要素・head要素・body要素・colgroup要素・tbody要素は、開始タグ・終了タグともに省略できる。この5種類の要素はしっかり暗記する

7.クッキーは1つ4KB、1サーバに20個、トータル300個まで保存できる。

クッキーには有効期限を設定することができ、有効期限を超えると保存されていたクッキーは無効になる。有効期限が指定されていると、有効期限内はブラウザを終了しても保存されていますが、指定されていない場合ブラウザの終了とともに削除される。

8.ディレクトリトラバーサル

Webサーバへの脆弱性攻撃の1つで、ブラウザから本来アクセスできないはずのサーバのファイルにアクセスする攻撃で別名「../攻撃」とも呼ばれる。

9.Webアクセス高速化

HTTPヘッダーフィールドの内、以下の設定でWebアクセスの高速化を設定できる。

- Cache-ControlはGeneral headerの1つで、Cahteを制御する。更新されていない情報についてキャッシュを利用させることでサーバの負荷軽減とWebページの表示高速化を期待できる。
- ExpiresはEntity headerの1つで、ページの有効期限以前であれば、キャッシュを利用させるので、サーバの負荷軽減とWebページの表示高速化を期待できる。
- Last-ModifiedはEntity headerの1つで、コンテンツが前回アクセス時から更新されていない場合、キャッシュを利用するので、サーバの負荷軽減とWevパージの表示高速化を期待できる。

10.HTTP,HTTPSプロトコル

HTMLのフォームで指定できるのはGETとPOSTのみである。

GETでは、リクエストパラメータはURLに含まれるが、POSTではボディに含まれる。

GETではデータサイズの制限があるが、POSTにはない。

GETリクエストに対しては、ステータスコード付きのレスポンスメッセージが戻る。

11.Ajax

Ajaxは従来のHTMLからの大幅な発展に貢献した技術の1つ。従来はクライアントからハイパーリンクをクリックするなどの対話型のリクエストに応じる形でサーバがコンテンツを送信していた。

Ajaxの登場で非同期にデータを送受信することが可能になり、よりリッチなアプリケーションへと発展した。AjaxはXMLHttpRequestをJavaScriptを介して利用することで実装する。実際にはより実装しやすくしたjQueryだけでなくJSONも利用できる。

12.カスタムデータ

data-の後の文字列の制限は、XMLの命名規則に加え、下記の3つの制限がある。
- XMLとして使用する場合であっても、名前をxmlで始めてはならない。
- 名前にコロン(:)を使ってはならない。
- 名前に大文字のA～Zを含めてはならない。

ハイフン(-)、アンダースコア(_)は使用可能。

13.円マーク(&yen;)

円マークは以下の表記で正しく表示できる。


Aは、文字実体参照と呼ばれ、人が理解・識別しやすい。

B・Cは、数値文字参照でBは10進数表記、Cは16進数表記となる。8進数表記では正しく表示できない？

ちなみにバックスラッシュを確実に表記するのは以下

```html
<span style="font-family:arial">\</span>
```

14.link要素

hreflang属性は、"en", "en-us", "fr-ca" などの形式になる。
media属性は、携帯デバイスの場合、"handheld"を使用する。
rel属性の"canonical"はSEOへの重複ページの影響を回避できる。

15.画像の形式

- BMP(Bitmap Image)は、Windowsに作られた画像フォーマットで、Webでは使用できない。
- JPEG(Joint Photograhic Experts Group)の特徴は、フルカラーの表現ができ、グラデーションも綺麗に表現する。
- PNG(Portable Network Graphics)は、JPEGと同様、フルカラーの表現ができるが、ファイルサイズがJPEGより大きくなる場合がある。JPEGと異なり、背景の透過が可能。
- GIF(Graphics Interchange Format)は、背景の透過が可能で、アニメーションも可能。
- TIFF(tagged Image File Format)は、高解像度の画像に向いている。

16.認証

HTTPでは、BASIC認証とDigest認証の2種類の認証方法がある。

BASIC認証では、ユーザが入力したユーザ名とパスワードは一切暗号化されることなく送信される。

Digest認証では、ユーザ名とパスワードは、MD5というハッシュ関数を使用して送信する。※ダイジェスト化ともいう。

BASIC認証をホスティングサービスを利用して実装しようとしたが、無料枠だと難しい。。

17.titleタグ

HTML5より前のHTNL/XHTMLでは、title要素は1つの文書に必ず1つ必要だったが、HTML5では「iframe要素のsrcdoc属性で指定する文書」および「上位レベルのプロトコルがタイトル情報を提供している場合」においてはtitle要素を省略することが可能。また、title要素自体は条件によって省略可能ですが、開始タグ・終了タグともにタグの省略は一切できない。


18.Data URI

Data URIは、通常外部ファイルへのURIを記述する、imgタグのsrc属性やCSSのurl()に直接画像データなどを埋め込むための技術。

Data URIはデータの場所を指すものでない。

20.HTTPS

HTTPSはWebクライアントとWebサーバの間の通信を暗号化するほかに、Webサーバの正当性を保証する仕組みを持っている。正当性を保証するために、Webサーバにはサーバ照明書と呼ばれるファイルを設置する。サーバ証明書には、Webサーバのホスト名などの情報の他、正当性を保証するための第三者機関による認証局による電子署名が付与されている。

21.Base64

- バイナリデータ(不可視文字を含む)を印字可能文字を使って表現する仕組み。
- Base64ではA-Z,a-z,0-9,+,/の64文字
- 元のデータが24ビットで割り切れない場合は、足りない分を'='で埋める。
- 変換後の文字列が64文字以上になった場合は、64文字ごとに改行文字を挿入する。
- データ量は変換前と後で変わる。

22.外部stylesheetの読み込み

HTML標準では、link要素には必ずrel属性を指定する必要がある。type属性が省略されると、rel属性の値が「stylesheet」の場合のデフォルト値「text/css」が適用される。

22.xhtml

- html要素にはデフォルト名前空間として以下のようにxmlns属性を指定する必要がある。
- xhtml要素という要素はいずれのXHTML仕様にも存在しない。

```html
<html xmlns="http://www.w3.org/1999/xhtml" xml:lang="ja" lang="ja">
```

- 要素名や属性名は全て小文字で記述する必要がある。
- 属性の値は「"」や「'」で囲む必要がある。
- 空要素には終了タグを付加するか、開始タグの閉じカッコを「/>」としなければならない。

23.Expiresヘッダーフィールド

- キャッシュを利用することでウェブページに関するロードの高速化が期待できる。
- HTTPレスポンスに付与される。
- キャッシュの有効期間期限に指定できる。
- スクリプトやスタイルシートにも利用できる。
HTTP/1.1において、Cahe-Controlフィールドのmax-age指示子がある場合は、Expiresフィールドは上書きされる。

24.hgroup要素

hgroup要素の子要素は次のように配置する仕様となっている。
p要素と0個以上 + h1~h6要素のうち1つ + p要素を0個以上

```html
<hgroup><h1>見出し</h1></hgroup>
<hgroup><h1>見出し</h1><p>副題</p></hgroup>
```

25.placeholder属性

- 入力のための短いヒントを表示させる属性である。
- 一般に値には入力礼や書式の説明などが指定される。
- ユーザが文字を入力すると値は表示されなくなる。
- この属性をラベルの代わりとして使用すべきではない。

26.メディア要素

- HTMLのメディア要素とはvideo要素とaudio要素のことである。
- video要素のsrc属性には音声データ(mp3形式など)を指定できる。
- audio要素のsrc属性には映像データ(mp4形式など)を指定できる。

27.br要素

HTML Living Standardの仕様書において、br要素の正しい用途の例として記載されているのが「詩」と「住所」の表記における改行。

「br要素は1つの段落内で項目を分けるために使用してはならない」ため、以下は悪い例

```html
<p>
<label>名前：<input name="nm"></label>
<br>
<label>住所：<input name="ad"></label>
</p>
```

正しくは以下のように書く。

```html
<p>
<label>名前：<input name="nm"></label>
</p>
<p>
<label>住所：<input name="ad"></label>
</p>
```
